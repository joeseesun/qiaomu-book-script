# Qiaomu Voice Benchmark

Use this as the default quality target for book spoken scripts.

The model is not "cover every theme". The model is:

```text
strong author/book anchor -> one uncomfortable thesis -> 3-5 concepts -> plain logic -> ordinary examples -> calm branded closing
```

For recurring book recommendation videos, default opening can be:

```text
大家好，我是向阳乔木，又到好书推荐时间。
```

## What The User's Edited Version Teaches

### 1. Open With A Cognitive Anchor

Prefer a quick contrast or forgotten-author hook.

Good pattern:

```text
弗洛伊德你可能听过。
荣格你可能也知道。
但还有一个人，和他们并称心理学三巨头，但却在很长一段时间里，被普通读者忽略了。
他叫阿尔弗雷德·阿德勒。
```

Why it works:

- It gives the viewer a known map before introducing the book.
- It makes the author worth hearing without a Wikipedia biography.
- It creates instant contrast: famous people everyone knows vs. a neglected thinker.

### 2. State The Book's Hard Thesis In Simple Words

Prefer:

```text
弗洛伊德说，你现在的样子，和过去有关。
阿德勒说：过去发生了什么很重要，但更重要的是，你现在怎么看过去。
```

Avoid making the thesis sound like blame.

Do not overstate it as:

```text
你的不幸都是你自己选的。
```

The stronger, safer framing is:

```text
过去发生了什么很重要，但更重要的是，你现在怎么看过去。
```

### 3. Explain Each Concept With One Clean Logic Chain

Use this structure for every core idea:

```text
概念名。
一句话解释。
三个生活例子。
一句真正的结论。
```

For 《被讨厌的勇气》:

- 目的论：我们会用过去、自卑、受伤经历保护自己，避免开始新生活。
- 课题分离：谁承担后果，谁拥有课题。
- 被讨厌的勇气：不能把“不被讨厌”当成目标。
- 共同体感觉：我和他人有关，我在对他人有用。

### 4. Examples Should Be Ordinary And Immediately Recognizable

Use examples like:

- 小时候总被大人否定，所以现在没有自信。
- 曾经受过伤，所以不敢谈亲密关系。
- 能力不够，所以不敢争取机会。
- 给领导发消息，写完又删，删完又改，连标点都要揣摩。
- 拒绝朋友一次，却要解释十遍，生怕对方生气。
- 参加不想去的饭局，怕别人说你不合群。
- 删掉发出去的内容，因为回应太少。

Do not drift into extreme trauma, workplace spectacle, or moral instruction unless the user asks.

### 5. Keep The Language Slightly Milder Than The Sharpest Version

Prefer:

```text
这不是否认伤害，伤害肯定会有。
但阿德勒提醒我们，伤害不能影响你的人生主动权。
```

The tone should be direct, but not punitive.

Avoid sounding like the viewer is weak, guilty, or fully responsible for being hurt.

### 6. Use Repeatable Lines

Good lines from the benchmark direction:

```text
这些时刻，本质上都是你在用自由换认可。
```

```text
被讨厌的勇气，不是变得强硬。
```

```text
这不是失败。
这是自由的正常代价。
```

```text
真正的自由，不是让所有人都喜欢你。
真正的自由，是你终于不用为了被喜欢，而委屈自己。
```

### 7. Close Without Overexplaining

Preferred ending when no platform CTA is required:

```text
感谢你的收看，我是向阳乔木。
```

Do not keep explaining after the final takeaway.

## Micro-Revision Principles Learned From The User Edit

The user's edit tightened the draft in a consistent direction. Apply these rules in future scripts.

### A. Be Direct Without Sounding Punitive

Prefer measured challenge over harsh narrator judgment.

Better:

```text
这个观点放在今天，依然很难被很多人接受。
```

Avoid:

```text
这个观点放在今天听，依然很刺耳。
因为它不让你一直躲在“我没办法”后面。
```

The user tends to delete lines that sound like the speaker is diagnosing or scolding the viewer.

### B. Prefer "怎么看" Over "怎么使用" When Talking About Pain

For trauma, past experience, regret, failure, or原生家庭 topics, avoid wording that sounds like the viewer is manipulating their pain.

Prefer:

```text
过去发生了什么很重要，但更重要的是，你现在怎么看过去。
```

Use "使用" only when the context clearly discusses a psychological mechanism and the tone is already safe.

### C. Use One Concrete Word To Make A Sentence Visible

Small additions can make a sentence more vivid without making it longer.

Better:

```text
小时候总被大人否定。
```

Weaker:

```text
小时候总被否定。
```

The user prefers ordinary specificity over abstract generality.

### D. Keep Concept Terms Stable

Do not over-paraphrase important book concepts.

Better:

```text
但不要总替别人承担课题。
```

Weaker:

```text
但不要替别人活完他的反应。
```

The second line is more literary, but the first line keeps the book's concept alive and is easier to remember.

### E. Delete The Third Example When Two Are Enough

If two examples already make the concept obvious, stop.

The user deleted the parent-life-defense example from the课题分离 section. That suggests a preference for momentum over exhaustive resonance.

Rule:

```text
Use 2-3 examples per concept. Stop at 2 when the point is already clear.
```

### F. Cut Meta-Commentary

Avoid speaker comments that explain how heavy, sharp, or important a sentence is.

Often delete:

```text
这句话很重。
所以你看。
如果你只记住一句话，我希望是这句。
```

Let the idea carry itself.

### G. Prefer Common Oral Words Over Stylish Words

Better:

```text
这种感觉，比被夸更好。
```

Weaker:

```text
这种感觉，比被夸更稳。
```

"更稳" is not wrong, but "更好" sounds more ordinary and less like creator jargon.

### H. End On The Cleanest Emotional Landing

The preferred final turn:

```text
真正的自由，不是让所有人都喜欢你。
真正的自由，是你终于不用为了被喜欢，而委屈自己。
感谢你的收看，我是向阳乔木。
```

Avoid adding another broad maxim after this. It weakens the landing.

## Micro-Revision Principles Learned From The Pyramid Principle Edit

The user's 《金字塔原理》 edit shows preferences for business/tool-book scripts. Apply these when the book teaches a method, framework, communication skill, productivity system, management idea, or consulting-style thinking.

### I. Use A Column-Like Opening For Recurring Recommendations

Prefer:

```text
大家好，我是向阳乔木，又到好书推荐时间。
```

This sounds more like a recurring program than:

```text
大家好，我是向阳乔木，今天我给大家推荐一本书。
```

### J. Make Workplace Dialogue More Colloquial

Prefer natural spoken workplace lines.

Better:

```text
你做了三十页 PPT，讲了十分钟，大家问：你到底想说啥？
```

Weaker:

```text
你做了三十页 PPT，讲了十分钟，对方只问一句：所以你到底想说什么？
```

The user prefers shorter, more everyday speech over polished paraphrase.

### K. Diagnose The Problem More Precisely

For tool books, identify the failure mode with practical precision.

Better:

```text
很多聪明人不是没有想法，而是想法太散，没有结构和顺序。
```

Weaker:

```text
很多聪明人不是没有想法，而是想法没有顺序。
```

"太散，没有结构和顺序" explains the real business problem better than "没有顺序".

### L. Compress Setup Phrases

Prefer short oral compression.

Better:

```text
曾在麦肯锡工作。
先给答案。
听起来不复杂，四个词就能概括。
```

Weaker:

```text
曾经在麦肯锡工作。
先把答案给出来。
这套结构听起来很简单。
```

### M. Combine Lines When The Relationship Is Tight

Separate lines are good, but do not split a cause-result sentence when combining improves flow.

Better:

```text
后面的信息，不再是一堆材料，而是变成了支撑结论的理由。
```

Weaker:

```text
后面的信息，不再是一堆材料。
它们都变成了支撑结论的理由。
```

This also means `不是……而是` / `不再……而是` should not be treated as absolutely forbidden. Use it when it makes a clean contrast.

### N. Reduce Hedging When The Logic Is Clear

Better:

```text
它可能重要，但不支撑这个结论。
```

Weaker:

```text
它可能重要，但不一定支撑这个结论。
```

If the example is designed to show a category error, be decisive.

### O. Prefer User-Centered Care Over Moral Obligation

Better:

```text
它教你照顾听众的大脑。
大脑是人最耗能的器官之一，最怕整理散乱的内容。
```

Weaker:

```text
它教你尊重听众的大脑。
别人没有义务替你整理一堆散乱材料。
```

The user prefers helpful explanatory framing over scolding or obligation language.

### P. Use Business Verbs That Sound Native

Prefer:

```text
再把理由码齐。
如果必须本周启动，建议先做核心流程。
还不知道你站哪边。
```

Weaker:

```text
再把理由摆整齐。
如果必须本周启动，只能先做核心流程。
还不知道你站在哪边。
```

The user's version sounds more like real workplace speech.

### Q. Prefer Clear Thinking Language Over Metaphor

Better:

```text
它把表达这件事讲得很透。
脑子里的结构没搭好。
自己也还没想明白。
```

Weaker:

```text
它把表达这件事讲得很诚实。
脑子里的结构还没有搭好。
自己也还在雾里。
```

For business/tool books, avoid poetic abstraction. Say the practical thing.

## Reusable Script Skeleton

```text
大家好，我是向阳乔木，今天我给大家推荐一本书。

[熟悉人物/流派/常识锚点。]
[被忽略的人或反常识观点。]
[为什么这本书值得讲。]

[核心概念一。]
[一句话解释。]
[生活例子 1。]
[生活例子 2。]
[结论。]

[核心概念二。]
[判断标准或操作方法。]
[生活例子。]
[边界说明。]

[核心概念三。]
[书名或核心命题的真正含义。]
[生活例子。]
[自由或代价。]

[可选核心概念四。]
[把锋利观点带回关系和生活。]

[全书逻辑一句话收束。]
[最终可复述金句。]

感谢你的收看，我是向阳乔木。
```

## Quality Bar

A good draft should be:

- short enough to read in one take
- clear enough that every paragraph answers "so what"
- memorable enough that at least one sentence can be clipped as a title
- restrained enough that it does not sound like lecturing the viewer
- concrete enough that examples feel like daily life, not invented drama
- plain enough that the speaker never sounds like they are showing off a phrase
- compressed enough that the user does not need to remove explanatory scaffolding
